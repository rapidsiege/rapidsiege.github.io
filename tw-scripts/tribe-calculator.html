Changelog

Feature and change history for the Tribe Calculator, per version bump.

v1.7.42026-06-14
  • Tribe Timings — grey out cells that miss the deadline: with an “Arrive by” deadline set, each per-unit travel/arrival cell whose unit can’t land in time is dimmed (in addition to hiding villages where nothing makes it), so you can see at a glance which units of a kept village still arrive.
  • Tribe Timings — Export Per-Player BB: new button that groups the shown rows by player (like Plan Offensive’s per-player export), so each member can be told exactly which of their villages reach the target and when (arrival time, pace unit, rally link).
  • Tribe Timings — target village excluded: a village sitting on the target coordinate is no longer listed as a sender to itself.
  • Tribe Timings — off-power tier badge: the Off Power column now shows the Complete Off / 3/4 / 1/2 tier badge next to the value (offensive mode).
  • Database updated time in three clocks: the web-mirror “updated” line now shows the timestamp in UTC, your browser’s local time, and game-server time (UTC + the configured offset).
v1.7.32026-06-14
  • Corrected morale formula: now morale = 3 × defender points ÷ attacker points + 0.3, clamped between 30% and 100% — fitted exactly to in-game battle simulations (the previous + 0.25 was off by ~5 points across the mid-range).
v1.7.22026-06-14
  • Morale column in Tribe Timings (Offensive mode): shows each attacker’s morale against the target, using morale = 3 × defender points ÷ attacker points + 0.25 (capped at 100%). Points are each player’s total village points aggregated from the loaded map database; shows “—” until the DB is loaded.
v1.7.12026-06-14
  • Shortened the “Total Def Power” header to “Def Power” across the Rankings, By Player and Villages tables (fixes the Rankings horizontal scrollbar).
  • Added this Changelog tab.
v1.7.02026-06-14
  • Swordsman defense fix: corrected the data (50 vs infantry / 25 vs cavalry). Every Defensive Power figure for armies containing swords was previously wrong.
  • Combined Defensive Power: infantry + cavalry defense are now shown as one value (with an “X inf + Y cav” tooltip) on Overview, By Player, By Villages and Rankings; the Overview bar splits into infantry and cavalry segments.
  • Sortable Villages table: click any column header to sort (the old Sort dropdown was removed).
  • Wired unit/stat icons throughout the Overview, both data tables, Tribe Timings and Rankings.
  • Banked the full reverse-engineered TW battle-combat formula spec for a future Battle Simulator.
v1.6.32026-06-13
  • Tribe Timings “Arrive by” deadline (date + time): hides villages that can’t land in time, adds an “Any” pace option, and shows arrivals in server time.
  • Tier badges replace plain text for off tiers in Offensive Targets and Settings.
  • Off tier auto-bump: when a requested off tier has no candidates left, the planner upgrades to the nearest stronger tier and warns.
v1.6.22026-06-12
  • New targets default their Snob Mode to Solo.
v1.6.12026-06-12
  • Off min/max distance band — keep close offs free for a quick second round.
  • Server time setting (UTC offset) with a live server clock.
  • Launch dates on plan windows + a past-launch feasibility guard that drops or flags offs that would have to launch in the past.
v1.6.02026-06-12
  • Server URL setting and rally-point links in the Plan table.
  • Configurable default off counts per new target.
  • Export Per-Player BB — one forum block per sender.
  • Split-off snob icon in the BB exports.
  • Light cavalry & spies now counted in defensive power.
  • “Send with” pace-unit order links in Tribe Timings.
  • Tribe names from ally.txt + a sortable Database table with a Tribe column.
v1.5.22026-06-12
  • Snob travel range limit (default 70 fields).
  • “Escorted” renamed to “Split Off”.
  • Fix-time button glyph changed to “=”.
v1.5.12026-06-12
  • Allow multiple noble trains from the same village.
v1.5.02026-06-12
  • Start/end time-input window editor for off and snob windows.
  • Per-target multiple off windows with counts.
  • Per-sender noble counts.
  • Tier-ordered landings — Completes land earliest, then 3/4, then 1/2.
v1.4.02026-06-12
  • Added the offensive-planning tabs: Tribe Timings, Offensive Targets, Plan Offensive and Database.

v1.0.x – v1.3.x — initial releases (predate version tracking).
