🚀 First Setup

1 Download the Village Database

You will need the basic village.txt and player.txt files. These can be downloaded from the game's public information files. For ES100 as an example:

Open es100.guerrastribales.es/map/village.txt in your browser, then right-click → Save As and save it as village.txt in the same folder as this HTML file.

Do the same for es100.guerrastribales.es/map/player.txt → right-click → Save As → save as player.txt in the same folder. These files give you village names, IDs and player data.

2 Connect the Village Database

Go to the Village DB tab → click Connect DB Folder → select the folder containing village.txt and player.txt. The folder is remembered for future sessions.

If you download updated map files later, click the button again to reconnect, or re-select the folder if needed.

3 Configure world settings

In the top bar, set World Speed and Unit Speed to match your server. For es100: World Speed 2×, Unit Speed 0.5×. Set My Player to your in-game name — this filters auto-generation to only your assigned requirements.

4 Import your villages

Option 1 — Script (automatic): Drag the gold bookmarklet button to your browser's bookmarks bar. Open TW → Recruit → Mass Recruit tab, click the bookmark — your troops are copied to clipboard. Come back here and click 📋 Paste Villages. This script only reads troop data already visible on the page — it does not interact with the game or send anything to the server.

Option 2 — Save HTML manually: Open TW → Recruit → Mass Recruit tab, press Ctrl+S → Save As → HTML Only. Back here, click 📄 Select HTML File and pick the saved file — villages are imported automatically. This is the safest option: no script runs on TW at all — you are simply feeding the saved HTML to the planner so it can read your units.

Use 🗑 Clear Villages to remove all villages and start fresh.

5 Add your villages (manual)

In My Villages, click + Add Village and fill in the coords, village ID, and troop counts (Axes, LC, Rams, Catapults, Nobles) manually.

🚀 Configuración Inicial

1 Descargar la Base de Datos

Necesitarás los archivos básicos village.txt y player.txt. Estos se pueden descargar desde los archivos de información pública del juego. Para ES100 como ejemplo:

Abre es100.guerrastribales.es/map/village.txt en tu navegador, luego clic derecho → Guardar como y guárdalo como village.txt en la misma carpeta que este HTML.

Haz lo mismo con es100.guerrastribales.es/map/player.txt → clic derecho → Guardar como → guárdalo como player.txt en la misma carpeta. Estos archivos proporcionan nombres de pueblos, IDs y datos de jugadores.

2 Conectar la Base de Datos

Ve a la pestaña Base de Datos → haz clic en Conectar Carpeta BD → selecciona la carpeta con village.txt y player.txt. La carpeta se recuerda para futuras sesiones.

Si descargas archivos de mapa actualizados más adelante, vuelve a hacer clic en el botón para reconectar.

3 Configurar el mundo

En la barra superior, ajusta Velocidad Mundo y Vel. Unidades según tu servidor. Para es100: Velocidad 2×, Vel. Unidades 0.5×. Escribe tu nombre en Mi Jugador — esto filtra la generación automática a solo tus requisitos asignados.

4 Importar pueblos

Opción 1 — Script (automático): Arrastra el botón dorado a la barra de marcadores. Abre GT → Reclutar → Reclutamiento Masivo, haz clic en el marcador — las tropas se copian al portapapeles. Vuelve aquí y haz clic en 📋 Pegar Pueblos. Este script solo lee los datos de tropas ya visibles en la página — no interactúa con el juego ni envía nada al servidor.

Opción 2 — Guardar HTML manualmente: Abre GT → Reclutar → Reclutamiento Masivo, pulsa Ctrl+S → Guardar como → Sólo HTML. De vuelta aquí, haz clic en 📄 Seleccionar archivo HTML y elige el archivo guardado — los pueblos se importan automáticamente. Esta es la opción más segura: no se ejecuta ningún script en GT — simplemente le estás dando al planificador el HTML guardado para que lea tus unidades.

Usa 🗑 Borrar Pueblos para eliminar todos los pueblos y empezar de cero.

5 Añadir pueblos (manual)

En Mis Pueblos, haz clic en + Añadir Pueblo e introduce las coordenadas, ID del pueblo y tropas (Hachas, Lijas, Arietes, Catapultas, Nobles) manualmente.

⚔ Planning an Op

1 Import targets

Go to Targets → click ⚔ Off Targets and paste the BB-code op plan from your tribe forum. Lines with [unit]ram[/unit] = complete off, [unit]axe[/unit] = 1/2 off, [unit]snob[/unit] = noble train.

The easiest way: open the message or forum post where you received the op instructions, click Quote, and paste the whole quoted text. Everything — off sizes, times, coords, targets, players — is auto-imported.

For fake-only targets, use 💨 Fake Targets and paste one coordinate per line (e.g. 557|555). You can also add a target manually with + Add Target — set the type (Off / Fake) and requirements directly.

Use 🗑 Clear Targets to remove all targets and start a new op.

2 Auto-generate attacks

Go to Attack Plan → click ⚡ Auto-Generate. Pick a landing date, configure criteria (Power, Distance, Send Window) and options, then click Generate.

Attacks appear in the table sorted by send time. Any unassigned requirements (ran out of nobles or off villages) appear as red rows — review them and assign manually if needed.

3 Send your attacks

Click ⚔ Enviar on each row to open the TW rally point pre-filled with the correct troops and target. After sending, click ✓ Sent to mark it — the row moves to the bottom and greys out.

4 Add attacks manually

Click + Add Attack in the Attack Plan section header to add an individual attack. Select the source village, target, type (Off / Fake / Snob) and landing time.

5 Save & backup

Data auto-saves to browser storage. Connect a local file via ⟁ Connect File for persistent storage. Use ↓ Export / ↑ Import in the top bar for manual backups.

Use 📋 Export BB Code to copy the full attack plan as a BB-code table you can post directly in your tribe forum.

⚔ Planificar una Op

1 Importar objetivos

Ve a Objetivos → haz clic en ⚔ Obj. Off y pega el plan BB-code del foro de tu tribu. Líneas con [unit]ram[/unit] = off completo, [unit]axe[/unit] = 1/2 off, [unit]snob[/unit] = tren de nobles.

La forma más fácil: abre el mensaje o post del foro donde recibiste las instrucciones de la op, haz clic en Citar, y pega todo el texto citado. Tallas de off, horarios, coordenadas, objetivos y jugadores se importan automáticamente.

Para objetivos solo de fake, usa 💨 Obj. Fake y pega una coordenada por línea (ej. 557|555). También puedes añadir un objetivo manualmente con + Añadir Objetivo — establece el tipo (Off / Fake) y los requisitos directamente.

Usa 🗑 Borrar Objetivos para eliminar todos los objetivos y empezar una nueva op.

2 Auto-generar ataques

Ve a Plan de Ataque → haz clic en ⚡ Auto-Generar. Elige una fecha de llegada, configura los criterios (Poder, Distancia, Ventana de Envío) y opciones, y haz clic en Generar.

Los ataques aparecen en la tabla ordenados por hora de envío. Los requisitos sin asignar (sin nobles ni pueblos off disponibles) aparecen como filas rojas — revísalos y asígnalos manualmente si es necesario.

3 Enviar los ataques

Haz clic en ⚔ Enviar en cada fila para abrir la plaza de reuniones pre-rellenada con las tropas y el objetivo correctos. Tras enviar, haz clic en ✓ Enviado — la fila baja al fondo y se oscurece.

4 Añadir ataques manualmente

Haz clic en + Añadir Ataque en la cabecera del Plan de Ataque para añadir un ataque individual. Selecciona el pueblo de origen, objetivo, tipo (Off / Fake / Noble) y hora de llegada.

5 Guardar y hacer copia de seguridad

Los datos se guardan automáticamente en el navegador. Conecta un archivo local con ⟁ Conectar Archivo para almacenamiento persistente. Usa ↓ Exportar / ↑ Importar en la barra superior para copias manuales.

Usa 📋 Exportar BB Code para copiar el plan de ataque completo como tabla BB-code y pegarlo directamente en el foro de tu tribu.
